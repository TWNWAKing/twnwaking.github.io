    複雜度Complexity + 大O符號 Big-O-Notation

    為什麼我們需要複雜度?

    一個好的程式不僅是要寫得出來(廢話)，還要比效率，因為電腦就是來幫我們節省時間的，若電腦跑到天荒地老意義就不大了。

    複雜度in程式競賽

    程式設計競賽也會要求要在規定時間內跑完程式(通常是1~5秒內)，那我們想評估一個算法預期的時間就要想出一個量化的準則，而我們引入複雜度來作為一種分析準則，那我們可以根據分析出來的結果，來隨機應變選擇最適合的演算法。

    通常在競賽中，會比較常強調時間複雜度，因為時間寶貴嘛(??)。當然相應地，也有空間複雜度來評估預計執行時要消耗的資源，與時間來說是相對的(畢竟空間換取時間???)，(通常)不能又快又消耗不多資源，所以在實作中要時間與空間都取得平衡才能寫出最適合的程式。

    筆者OS: 通常比賽給的記憶體空間真的很足(除非刻意卡空間)，只要不要太誇張就好(像是開\(10^{18}\)長度的array，開110棵線段樹之類的)

    Big O notation

    我們引入大O符號Big \(O\) Notation。

    大O符號是有嚴謹數學定義的，但因為我的功力較弱(?)所以我將會以白話文來詮釋我的理解(因為暫時可以不用到那麼深的去了解它)

    白話文來說，\(O(f(x))\)像是一個非常粗略估計執行次數的函數，用來描述在輸入一個常數n，函數們之間的大小關係，而且函數只取增長速度的最快的項(所以才說粗略)。

    通常最基本和簡單的動作(加加減減，swap，比較之類的)會被算做"執行一次"。

    如果一個演算法有多種複雜度會取最差的Case。

    所以它比較像是一個悲觀的分析，例如\(n=10^5,\ 遇上O(n^2)\)不會過的機率非常大，後面會講為什麼(?)。

    像是最常見的Bubble sort如果一次就好了複雜度\(O(n)\)，反之如果全部逆序就變成\(O(n^2)\)了，時間複雜度會取後者。

    常見的時間複雜度

    • \(O(1)\)
    • \(O(n)\)
    • \(O(\log n)\)
    • \(O(n \log n)\)
    • \(O(n^2)\)
    • \(O(2^n)\)
    • \(O(\log \log n)\)
    • \(O(n!)\)
    • \(O(n \log^2 n)\)
    • ...

    注意這裡log是什麼底數不太重要，因為換底公式的關係讓他們最多只差常數倍，算是大O符號能容忍的誤差範圍內。(但是因為店考裡面一堆Binary的架構所以通常是指base 2)

    顯而易見當n很大的時候\(1< \log \log n < \log n < n < n \log n < n^2 < 2^n < n!\) ...，所以就可以比大小了!

    複雜度說明

    以下簡略說明(這裡以執行次數和時間)各個複雜度的意涵。

    不過舉例有點爛就是了(?)

    \(O(1)\)

    常數複雜度:不管輸入什麼值，程式執行次數都是固定的常數。 舉例:輸出 "Hello World" 十次。

    \(O(n)\)線性複雜度:輸入的值與執行次數成(類似)線性關係。 舉例:輸入n之後輸出 "Hello World" n次，線性搜尋(就是直觀地n個資料每個都對照一遍)之類的。

    \(O(\log n)\)

    次線性複雜度:就如同字面所示。(我常常會叫它對數級別的複雜度)

    舉例:與上方\(O(n)\)做對比，在\(n\)蠻大的時候，\(O(\log n)\)是非常非常快的，而人們常常用二分的觀念讓\(O(n)\)降到\(O(\log n)\)。

    \(O(n^2)\)

    舉例:出現兩個獨立且樸素的For loop常常就是\(O(n^2)\) 上面有說明過一個經典的就是Bubble Sort了。

    \(O(n \log n)\)

    舉例:事實上我們也可以把Sort壓到\(O(n \log n)\) (畢竟\(n^2>n \log n\))，像是Merge Sort(相應地，它就要比較多額外空間了) 跟\(O(n)\rightarrow O(\log n)\)的道理一樣，人們常常用二分的觀念讓\(O(n^2)\)降到\(O(n\log n)\)。

    \(O(2^n),O(n!)\)

    通常是枚舉(enumerate)會出現的，十分糟糕的複雜度。

    這樣各種演算法都有相應的空間和時間的估計量化了，所以我們會一直想辦法改進算法(策略)。

    最後來挖個坑: 如果我們有天是分而治之(暫時理解為把資料分成好幾塊同時處理)，那它的綜合起來的複雜度要怎麼計呢?

    這時候就要引入 : Master Theorem a.k.a 大師定理(X)來算複雜度(其實應該要翻成「主定理」)

    不過我太懶了這裡就不介紹了。
