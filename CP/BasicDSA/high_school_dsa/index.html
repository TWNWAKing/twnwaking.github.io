    基礎資料結構與演算法

    其實這部分是筆者的高中自主學習計畫，剛好有網站可以放此成果，那我就不客氣啦。

    我會把摘要、前言、心得與反思都放在此頁。

    以下為計劃書本體

    作者: 中崙高中 邱昱翔 (TWNWAKing)

    目錄

    請點開左側就有個單元的分類了喔!

    右側則是該文章的目錄。

    計畫

    計畫名稱: 初探基礎資料結構與演算法

    領域: 資訊領域

    動機與生涯探索:

    1. 對於資訊科學特別的有興趣，故想研讀一些關於此方面的知識。

    2. 想參與資訊相關競賽，故想學習這方面的知識。

    3. 未來預計往資訊相關科系發展。

    學習目標:

    1. 探索與學習到資訊科學中的資料結構以及演算法之相關知識。

    2. 能夠實際運用在資訊中包括參與CPE考試，Codeforces、Google Coding Jam和Facebook Hacker Cup演算法競賽等。

    3. 能夠在未來所遇到的問題中能運用它們的思維和工具來順利地解決問題。

    4. 期許自我資訊等此方面能力更上一層樓。

    學習資源:

    • Google

    學習設備:

    • 我的筆電

    預計時間表以及進度:

    以下的數字編號為週

    1. 複雜度+大O符號 + Array + Vector

    2. Stack + Queue + Linked List

    3. Searching

    4. Sorting

    5. Recursion

    6. Greedy

    7. Divide and Conquer

    8. DP

    9. Tree + Heap

    10. 製作成果(就是您眼前的網站)

    摘要

    作品形式

    文章

    我將以Markdown為主要架構，加上LaTeX所提供優美的數學式子做為內容的輔助。

    網站

    使用Mkdocs作為主要框架，搭配上介面超讚的Mkdocs Material主題所製作而成。

    伺服器的化是由Github Pages提供。

    內容概述

    內容以學習基礎的資料結構與演算法作為出發點，並以豐富的C++ Standard Template Library和豐富的網路資源為輔，來為進入資訊領域打下扎實的基礎。

    我在有些文章中間會穿插此知識與我想發展的領域-AI之間的連結。

    並且在每個知識點的最後我會學以致用，我會蒐集相關的應用以及嘗試使用它們來解一些Online Judge上的題目，增進實作能力。

    前言

    前來來放一下我會做這個的緣起好了。

    我覺得我會想要開始做的起因是因為我非常不滿意我高二時修課的表現。

    高二的時候，我修了 "進階程式設計" 的課程，但當時因為種種原因導致我的學習狀況不盡理想: 考試時的觀念題半猜半亂寫，實作題全空白。

    我還記得，當初的慘況:

    手寫題: 用表格模擬各種sorting algorithm的過程，完了，有些不會。

    實作題: 老師原本次要求做一個只要有push,pop,可讀取top跟輸出整個stack功能的Stack，腦中一片空白，想說完了，我怎麼都學不會。

    我記得後來都已經將考題的 Stack 改成C++ STL裡面的讓大家不用自己刻了，但還是做不出來，我還是對它們的實作一籌莫展。

    我發現就算在高一學過了基礎語法，但是我的實作能力還是接近0。雖然學了一些觀念，但完全不會實作應用，久而久之就忘了，不太OK。而且不實作我覺得是沒辦法發現一些在學習觀念時沒發現的盲點。

    我不甘心，我要學會它。

    And so the journey began!

    遇到的困難

    我覺得我中途遇到了兩個困難: 時間規畫不當+所需先備知識量不足

    先備知識不足

    我在開始學的時候才發現我高一所累積的的基礎程式能力根本不足以應付學習基礎資料結構所需的語法，例如struct, pointer, template等等的觀念我都是學了但不熟甚至是沒有看過的，所以在學習演算法與資料結構的途中，我不免地要去學習新語法，所以我覺得中途確實是有學的吃力的部分。

    解決辦法就是撥出額外的時間去慢慢的學。

    時間規劃

    承上點，我覺得時間規劃是有大問題的，我太小看學習這些知識所需要的時間了。

    本來我預估是每周三小時做學習，結果當然是完全不夠。

    然後原本預定最後一周做的學習成果發現我太多想講了，所以拖了超\(\sim\)級\(\sim\)久才做出來。

    也許以後遇到瓶頸時可以隨機應變，及時調整自我。(是否需要拉長時間?進度調整順序?)

    心得、反思以及期望

    心得與反思

    我將分成三點來說明。

    自學能力的展現

    我覺得我在自主學習之間成功了運用我的自學能力而自學成功(廢話)。

    而在在未來，我相信新東西的出現只會越來越快，所以顯得自學能力的重要。如果我有幸進入資工後，有了資工的扎實訓練再加上個人的自學，我相信我的學習效果會再翻倍。

    過程中發現我的一片天

    我覺得最大的收穫是我在自主學習資料結構與演算法過程中，在經過深入的學習後，我發現了我在資訊的 潛力與熱忱 ，我覺得這就是是我未來想要去發展的領域。

    再自主學習完成，之後我便開始反思我的志向是否就是資訊工程。最後我在查閱來自各方的資料以及評估自我後，我發現我自身不錯的 自學力和解題力 都是對於適合資工的特質，再加上我在學習基礎資料結構與演算法中我發現這些東西我蠻喜歡的，便確立了志向。

    學習過程中的額外收穫

    首先當然就是在過程中"被動地"對C++有更深刻的了解，這真的是獲益良多。

    也在途中發現程式競賽真的蠻有趣的，固有了參與ICPC競賽的想法。

    再來是我在做成果的時候為了排版去精通Markdown語法，為了打數學式子而去弄LaTeX，還有為了架Blog而去做有關於web的東西，我覺得這些都蠻有趣的。

    期望我能在這條路上走得長遠。

    期望

    在自主學習完之後的期望

    我在這個自主學習成果的每篇文章中或多或少有一些不甚理解的地方，例如物件導向，但我相信學習是永無止盡的，我希望我能繼續地學習它們。

    並且我覺得雖然我目前會那麼一點點的實作了，可是我還是不自滿，我希望透過是透過一定程度的刷題來增進我的技巧，並在過程中持續的檢討和檢視自己的學習成效，

    縱使我目前在各大Online Judge還沒有突出的成績，但每次的練習都是我成長的養分。

    長遠目標: 參與ICPC競賽。

    我對於未來規畫的期望

    當然是期望能順利錄取資工系啦，以下我會說出我的四點原因。

    首先，我希望我能在那裡獲得十分扎實的訓練。因為老實說到現在的我其實都是一直自學的狀態，網路上的資料多所以可信度需要好好的確認，所以如果有錯可能就是比較晚才會發現。我相信在專業的資工培訓下我能一直在正軌上。

    又因為網路上的資料比較零碎，其實我自己也知道這長期下來對於一個純新手不是一個太好的學習方法。而我相信在資工的有系統性的培訓下，我能吸收到全面且扎實的理論，並了解背後的原理，在經過自我整理後能成為一套非常完整的知識系統，讓我不會停留在"皮毛"的地方。

    而且在資工系的專業教育下，我能變得與眾不同，變成不可被替代的資訊人才，所以我覺得我非資工不可。

    最後，我也期望我能在資工系中認識和結交到讓我一生獲益的良師益友，攜手共創更美好的未來。(這句話好像有點老套了XD)

    希望資工的各位先進能給晚生一個機會，謝謝!
