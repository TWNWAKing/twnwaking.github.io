    Array / 陣列 / 數組

    Array，又稱陣列或是數組，為一種十分十分十分常見的資料儲存結構。

    強調三次十分是因為真的是所有之中最常用的，真的太重要啦!

    概念

    陣列是一個裝同Type的東西的容器，有點像是有限項的數列，要指定它的大小，指定後不能改大小了，除非你用std::vector

    陣列的長相會像這樣: arr[index]=value;

    index就是第幾項，value就是那項的值。

    宣告

    C++的宣告方法有三種，以下假設T為資料型別，理論上array什麼都可以裝。N為指定陣列的大小，且假設我們要宣告的array名字叫做arr

    //C-style array:
T arr[N];
//Array in standard template library.
array<T,N> arr;
//Fixed size vector is same as array.
vector<T> arr(N);

    常見的T為int, float, char, string,...

    但是! 你要裝怪怪的東西也是沒問題喔! 像是你可以裝std::set, std::map, std::array(沒錯就是array裡面包array...)

    如果要多維振烈最簡單的就是 int arr[N][N][N]; ，std::array跟std::vector 比較複雜一點就先不講了。

    基礎用法

    • 存取元素: arr[i]

    沒啦

    沒啦???

    其實用法蠻多的(?)但是最常用的就真的是裝資料然後讀取它

    • 請注意array是從第0項開始! 也就是說可以存取arr[0], arr[1], ..., arr[N-1]，每個都代表一個值

    其他可見https://en.cppreference.com/w/cpp/container/array

    指定初始值

    int arr[N]={1,2,3...};
array<T,N> arr={1,2,3...};
vector<T> arr(N,初始值);

    用Fixed size vector好像就不行先用那樣指定初始值，但是一班狀況的vector<int> arr= { 7, 5, 16, 8 };這樣好像可以。

    筆者OS: 通常陣列的初始值用程式執行時再塞進去就好了(?)比較少先在Code裡面打好。

    像是

    for(int i=0;i<N;i++>){
  cin >> arr[i];
  //或是arr[i]=指定的value;
}

    之類的。

    陣列可以不要編譯的時候就指定大小嘛?

    就是以下這個情況，行不行?

    int n;
cin >> n;
int arr[n];

    注意到了嗎? 我的陣列長度n是要根據我的輸入來決定的，這樣其實直覺奇萊是很正常，但是仔細想想感覺又不太對(?)

    這個東西就是可變長陣列!

    本來C++的古老的標準不行，但是可能因為這個操作太直覺了(?)所以編譯器就開始支持了。

    參考:https://gcc.gnu.org/onlinedocs/gcc/Variable-Length.html

    Array的用途

    1. 需要用到array的時候，好啦認真說

    2. 需要 連續存取又快速查詢某個位子的資料 的時候會用到。

    3. 不想開一堆變數的時候。

    4. 當知道資料量有多少的時候。

    5. 我想要再強調一次是真的非常萬用的結構。

    動態開點的Array?

    譬如說我今天有個array a(a的長度\(\le 10^6\), \(0\le a_i\le k\))，我想統計所有元素中，每個數字出現了多少遍

    最簡單作法就是再開一個k那麼大的另一個陣列來統計。

    但若domain有到\(10^9\)(注意可以算一下它需要多少容量，反正比賽中開這麼大一定爆炸)，直接裂開了

    注意到問題在哪裡了嗎? 因為array是佔用連續的一段記憶體，而我們要統計的數字最多只有\(10^6\)個，所以一堆陣列的格子都浪費了

    而且如果今天a有負的項怎麼辦，開不下去啦!

    總的來說，我們需要一個 動態開點的array ，要什麼數字開什麼格子就好。

    How? 答案是用平衡樹std::set或是對index做Hashing std::unordered_map (介紹待補)

    當然如果你夠毒瘤，用cc_hash_table或是gp_hash_table 也可以。

    Index不是整數怎麼辦?

    譬如說我今天想統計貓跟狗有多少隻好了，我可以

    arr["cat"]=1;
arr["dog"]=3;

    嗎? 當然可以，方法就是上述的平衡樹std::set或是對index做Hashing std::unordered_map。

    為何不用Map代替Array

    疑，那為什麼map可以動態開點又有自由的index，為何還需要array呢?

    因為不管是哪種map的存取速度都遠，遠，遠慢於陣列 (也沒有到慢那麼多啦，但是如果存取元素很多次速度就真的有差了)，所以兩個算是在不同狀況下各司其職。

    但是固定size的vector跟陣列一樣快(再次推銷)。
