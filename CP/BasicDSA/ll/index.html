    Linked List

    相較於Array，Linked List是一種"串"的資料結構，就是把資料串成一長條的意思。

    原理是這樣的，我們開"盒子"，每個盒子裝兩個東西: Data value和The position of next data(暫時叫它箭頭好了)，所以它會記錄下個元素在哪。等等會解釋這樣設計的原因。

    其實Linked List比起Array的"缺點"蠻多的，但是主要是有一個點是array完全比不上的，這也許才是它出現的原因吧(?)

    Array "失敗"的地方

    今天我有一個陣列，我想在某兩個元素之間插入一個元素怎麼辦呢?

    Array的作法，把空隙右邊的元素通通往右移(假設沒有Out of bounds)，然後插入，複雜度\(O(Length\ of\ array)\)

    而Linked List只要把左邊的元素的箭頭指向要插入元素的位子，然後把新插入的元素的箭頭指向右邊的元素。

    整體來說在\(O(1)\)時間完成，太神啦 !

    盒子的結構

    首先我們想一下: 所謂的盒子要用甚麼東西來實現呢?

    我們要引入struct，介紹的話以後可能會補吧，暫時先把它理解為 把很多個資料包在一起 。

    像是我最常舉例的就是紀錄學生資料

    struct Student{ 
    string name;
    int age;  
    int class_number;
    double height;
    double weight;
};

vector<Student> arr;

    這樣我們就有一個vector，其中每個Element都有五個Data。

    Linked List主體

    我將會給出單向Linked list的Code

    以下Code來自這裡，畢竟我就是在那邊學的所以我只是照打來練習，代碼根本一模一樣。

    畢竟大家寫出來的都差不多。

    其實直接看那個Link就好了吧，比較清楚。

    struct Node{ //盒子
  int value;
  Node *next;
};

void Insert(int val, Node *p){ //插入(要判斷該鏈表是否為空以免發生錯誤)
  Node *node=new Node;
  node->value=val;
  node->next=nullptr;
  if(p==nullptr){
    p=node;
    node->next=node;
  }
  else{
    node->next=p->next;
    p->next=node;
  }
}
void Delete(Node *p) {
  p->value=p->next->value;
  Node *t=p->next;
  p->next=p->next->next;
  delete t;
}
