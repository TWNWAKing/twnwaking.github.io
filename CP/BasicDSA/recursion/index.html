    遞迴Recursion

    遞迴為一種一直呼叫自己，套娃的Function。

    什麼意思呢? 我先來一段偽代碼

    function recursion(T n){
  if(結束條件成立) return 某個東西或是結束;
  return recursion(...)
}

    就是會一直呼叫自己直到達到終止條件的Funciton。

    例題

    我會放一些經典例題以及我在比賽中寫過的題

    Fibonacci

    int F(int n){
  if(n<=1) return 1;
  else return F(n-1)+F(n-2);
}

    However，這個方法是爛的，當\(n=100\)之類的遞迴深度會太深(呼叫自己太多次，爆炸)

    因為會重複算到重覆元素，故可以用DP大量縮減複雜度(變成線性)。

    當然，如果你夠毒瘤，可以用矩陣快速冪，這樣是對數等級的。

    GCD

    就是輾轉相除法的Implementation。

    有人證明出來複雜度是 \(O(\log \dfrac{\min(a,b)}{\gcd(a,b)})\)

    int gcd(int a,int b){
  if(b == 0) return a;
  else return gcd(b,a%b);
}

    線段樹建樹

    真毒瘤

    void build(int now=1,int l=1,int r=n){
    if (l==r) {segtree[now]=arr[l];return;}
    int mid=(l+r)>>1;
    build(now<<1,l,mid);
    build(now<<1|1,mid+1,r);
    segtree[now]=min(segtree[now<<1],segtree[now<<1|1]);
}

    (這段是筆者後來補上的，當時高二的我沒那麼毒瘤)

    河內塔

    https://cses.fi/problemset/task/2165/

    蠻經典的遞迴練習，遞迴關係如下:

    設有A,B,C三根柱子，n個圓盤，要從A移到C

    1. 我們要把A最上面的n-1盤移到B

    2. A的最大盤移到C

    3. B的n-1片移到C

    而當n-1=k的時候就會一直遞迴下去。

    而程式的精華在於遞迴函數move

    n表示我們現在在處理n個圓盤的Case

    a,b,c表示我從a柱移到c柱，b柱是暫放的。

    還蠻妙的

    一部分的Code 
    void move(int n,int a,int b,int c){
    if(n==1){
    cout << a << " " << c << "\n";
      }
    else{
    move(n-1,a,c,b);
    cout << a << " " << c << "\n";
    move(n-1,b,a,c);
    }
}

    Apple Division

    https://cses.fi/problemset/task/1623

    這題是CSES Problem Set裡面的其中一題，當時我知道這題是枚舉但是我當時連枚舉的程式都不會寫，忘記在哪看到這精湛的遞迴解，覺得很酷。

    就是一直枚舉如果第\(n\)個物品放到第1 or 第2堆的狀況，然後取差的絕對值最小的。

    雖然看起來是遞迴但是它的複雜度還是指數級的XD，本質還是枚舉。

    一部分的Code 
    int solve(int i,int s1,int s2){
  if(i==n) return abs(s1-s2);
  else return min(solve(i+1,s1+arr[i],s2),solve(i+1,s1,s2+arr[i]));
}
