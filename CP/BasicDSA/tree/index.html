    樹/Tree(簡介)

    本文將概述一些關於樹的觀念。

    概述+我的小看法

    樹這個Graph其實應該要從圖論開始學起，再去介紹它。

    可是在資料結構中，樹形的資料結構是常常會出現的(像是各種平衡樹)，縱使我們不知道任何樹的演算法(像是最小生成樹，樹鏈剖分)

    而我也是因為在資料結構中先接觸到樹(算是先偷用觀念?)，在深入了解後才發現樹也是一門十分大的學問，但這部分其實不在我的自主學習計畫的規畫內了，再深入真的是學不完，進度會落後。

    所以其實在資料結構中只要先知道樹的"形象"就好了，至於之後詳細的更圖論的內容，我會再補在這個網站上的某處，但就不是在此自助學習的計畫內了。

    期望未來能補上相關更深更廣的知識，讓我對它有更深層的了解。

    Binary Tree in AI

    決策樹!

    Binary Search Tree

    其中又以Binary Search Tree(BST)為一個蠻重要的Data Structure。就是設定一種準則讓Data經過每個節點資後會掉進左邊or右邊。

    因為理想的樹高大概是\(O(\log n)\)所以找東西很快(?)，在AI中因為通常有很多資料(?)，要快速尋找的話可以丟進BST儲存。

    但因為"如何將資料妥善的，均勻的散佈在左右子樹"是一個大哉問，這樣才能確保複雜度在\(O(\log n)\)等級的。

    因為避免所有東西都掉進同一邊的Tree這樣就退化成一條Chain了，要找資料就是線性複雜度，非常糟糕。

    常見的Binary Search Tree有

    • Treap

    • Splay

    • 紅黑樹

    • AVL Tree

    • 笛卡爾樹

    • 替罪羊樹

    • KD樹(高維度的BST)

    其中又以Splay跟Treap廣受競賽選手喜愛，因為靈活度高且代碼比起紅黑樹來說短太多了。
